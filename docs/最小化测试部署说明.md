# 最小化测试部署说明

这套测试配置只启动两个服务：

1. 一个 Embedding 模型服务 `qwen3-embedding-test`
2. 一个路由服务 `openai-router-test`

目标是先验证最小链路：

```text
client -> openai-router-test:18083 -> qwen3-embedding-test:41001
```

## 文件说明

- [docker-compose.test.yml](/home/dev/bxc/START_LLM/docker-compose.test.yml)：单模型测试版
- [docker-compose-openai-router.test.yml](/home/dev/bxc/START_LLM/docker-compose-openai-router.test.yml)：单路由测试版

## 配置说明

### 端口

为了尽量不和现有运行环境冲突，这套测试版改成了独立端口：

- 模型服务：`41001`
- 路由服务：`18083`

### 网络

这两个测试版 compose 共用一个自定义网络：

```text
start-llm-test-net
```

所以 `openai-router-test` 可以直接通过容器名访问模型：

```text
http://qwen3-embedding-test:30000
```

### 路由初始化

`openai-router-test` 在容器启动时会自动创建一个最小 `routes.db`，并写入：

- 模型名：`qwen3-embedding`
- 路由目标：`http://qwen3-embedding-test:30000`
- 别名：`qwen3-embedding`、`text-embedding-ada-003`

也就是说，这套测试版不需要你手工改 SQLite。

## 启动步骤

### 1. 启动单模型服务

```bash
docker-compose -f docker-compose.test.yml up -d
```

### 2. 启动测试路由服务

```bash
docker-compose -f docker-compose-openai-router.test.yml up -d
```

## 调用地址

统一入口：

```text
http://localhost:18083/v1
```

模型直连地址：

```text
http://localhost:41001/v1
```

## 最小验证示例

```python
from openai import OpenAI

client = OpenAI(api_key="EMPTY", base_url="http://localhost:18083/v1")

resp = client.embeddings.create(
    model="qwen3-embedding",
    input=["你好", "测试 embedding 服务"],
)

print(len(resp.data))
print(len(resp.data[0].embedding))
```

## curl 验证示例

### 1. 查看路由层可用模型

```bash
curl http://localhost:18083/v1/models
```

### 2. 通过路由层调用 Embedding

```bash
curl -X POST http://localhost:18083/v1/embeddings \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3-embedding",
    "input": ["你好", "测试 embedding 服务"]
  }'
```

### 3. 通过模型别名调用 Embedding

```bash
curl -X POST http://localhost:18083/v1/embeddings \
  -H "Content-Type: application/json" \
  -d '{
    "model": "text-embedding-ada-003",
    "input": ["你好", "测试 embedding 服务"]
  }'
```

### 4. 直连模型服务调用 Embedding

```bash
curl -X POST http://localhost:41001/v1/embeddings \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3-embedding",
    "input": ["你好", "测试 embedding 服务"]
  }'
```

## 当前默认假设

这套测试版默认沿用了仓库里的现有模型目录：

```text
/home/dev/model/Qwen/Qwen3-Embedding-0___6B/
```

如果你的测试机器没有这个模型目录，需要改 [docker-compose.test.yml](/home/dev/bxc/START_LLM/docker-compose.test.yml) 里的模型路径。

## 可调参数

这套测试版预留了几个环境变量，方便你不用改 yml 也能调整：

- `TEST_MODEL_PORT`
- `TEST_ROUTER_PORT`
- `TEST_GPU_ID`
- `TEST_MODEL_DIR`
- `TEST_SERVED_MODEL_NAME`
- `TEST_GPU_MEMORY_UTILIZATION`

示例：

```bash
TEST_GPU_ID=1 TEST_MODEL_PORT=42001 docker-compose -f docker-compose.test.yml up -d
TEST_ROUTER_PORT=28083 docker-compose -f docker-compose-openai-router.test.yml up -d
```

如果显存仍然不足，可以直接再压低一点：

```bash
TEST_GPU_MEMORY_UTILIZATION=0.35 docker-compose -f docker-compose.test.yml up -d
```

## 清理方式

```bash
docker-compose -f docker-compose.test.yml down
docker-compose -f docker-compose-openai-router.test.yml down
```

如果要重置测试路由库：

```bash
rm -f data_test/routes.db
docker-compose -f docker-compose-openai-router.test.yml up -d
```
